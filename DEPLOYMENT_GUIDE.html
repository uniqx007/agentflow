Enable Google Cloud Billing

Required to use Cloud Run, Vertex AI, and AlloyDB

⚠️
Important: Cloud Run has a generous free tier (2M requests/month free). You will NOT be charged unless you exceed free limits. Enable billing to unlock the services.
  • 1️⃣
    Go to console.cloud.google.com → Sign in with your Google account
  • 2️⃣
    Click the top-left menu → Billing → Link a billing account (use a debit/credit card — you get $300 free credits as a new user)
  • 3️⃣
    Create a new project: Click dropdown at top → New Project → Name it agentflow-project
  • 4️⃣
    Enable APIs: Go to APIs & Services → Enable APIs and enable: Cloud Run, Vertex AI, Cloud Build, Artifact Registry

Install Required Tools

Google Cloud SDK + Python 3.11 + Docker

💡
Easiest option: Use Google Cloud Shell — it has all tools pre-installed. Go to console.cloud.google.com and click the >_ icon at the top right.

Option A — Use Google Cloud Shell (Recommended, Zero Setup):

bash

# Step 3: Install Python 3.11
# Visit: https://www.python.org/downloads/

# Visit: https://www.docker.com/products/docker-desktop/

Setup Google Cloud Project

Configure your project ID and enable required services

bash — Run these commands one by one
# Set your project (replace with your actual project ID)
gcloud config set project agentflow-project

# Set default region (choose one close to you)
gcloud config set run/region asia-south1   # Mumbai (closest for India)

# Enable all required APIs
gcloud services enable \
  run.googleapis.com \
  aiplatform.googleapis.com \
  cloudbuild.googleapis.com \
  artifactregistry.googleapis.com \
  secretmanager.googleapis.com

# Verify billing is active
gcloud beta billing projects describe agentflow-project
✅
The asia-south1 (Mumbai) region gives you the best latency if you're in India/APAC.

Deploy to Cloud Run

Build the container and deploy — one command does everything

🚀 Option A — Deploy directly from source (Easiest):

bash
# Make sure you're inside the agentflow/ folder
cd agentflow

# Deploy! Cloud Build will build the Docker image automatically
gcloud run deploy agentflow \
  --source . \
  --region asia-south1 \
  --allow-unauthenticated \
  --set-env-vars GOOGLE_CLOUD_PROJECT=agentflow-project \
  --set-env-vars GEMINI_MODEL=gemini-1.5-pro \
  --memory 1Gi \
  --cpu 1 \
  --min-instances 0 \
  --max-instances 10
⏳
Takes 3–5 minutes on first deploy (building the Docker image). Subsequent deploys are faster (~1 min).

🐳 Option B — Build Docker image manually then deploy:

bash
# Build and push Docker image to Artifact Registry
gcloud builds submit --tag \
  gcr.io/agentflow-project/agentflow:v1

# Deploy the image to Cloud Run
gcloud run deploy agentflow \
  --image gcr.io/agentflow-project/agentflow:v1 \
  --region asia-south1 \
  --allow-unauthenticated \
  --memory 1Gi

✅ After successful deploy, you'll see:

Troubleshooting Common Issues

Quick fixes for the most common deployment errors

  • ❌
    Error: "Billing account not found" → Go to console.cloud.google.com/billing and link a billing account to your project.
  • ❌
    Error: "Permission denied on Vertex AI" → Run: gcloud projects add-iam-policy-binding agentflow-project --member="serviceAccount:..." --role="roles/aiplatform.user"
  • ❌
    Error: "Container failed to start" → Check logs: gcloud run logs read --service agentflow --region asia-south1. Usually a missing env variable.
  • ❌
    Error: "API not enabled" → Run: gcloud services enable run.googleapis.com aiplatform.googleapis.com
  • ✅
    Tip: Always test locally first: pip install -r requirements.txt && uvicorn main:app --reload then visit http://localhost:8000
